README.arduino.md
1 Welcome to the **Keyglove** Arduino firmware source. This project is provided
2 in full source form so that you can make the most of a very powerful and
3 customizable platform, and (hopefully) documented in such a way that you can use
4 it with as little effort as possible out of the box. All software required for
5 use is freely available. The sketch itself is organized such that you will
6 normally only need to edit the fewest possible number of files in order to
7 achieve the functionality you want.
8 
9 # Prerequisites
10 
11 The Keyglove Arduino sketch currently works only with the **Teensy++ 2.0 from
12 PJRC** (or, I suppose, if you are really clever with an [AT90USB128x](http://www.atmel.com/devices/sam3x8e.aspx) MCU in your
13 own design). This board was chosen for its high I/O count, which is nice for
14 Keyglove designs with a large number of sensors. The [ATSAM3X8E](http://www.atmel.com/devices/sam3x8e.aspx) is planned for
15 the next revision and custom controller, but for now the T++2 is the best choice
16 and only supported platform.
17 
18 Additionally, you can optionally include certain modular components in your
19 chosen configuration, such as Bluetooth, motion, etc. Here is the hardware and
20 software that you should be looking at:
21 
22 ### Required components:
23 
24 1. [Teensy++ 2.0 AT90USB128x microcontroller board](https://www.pjrc.com/store/teensypp.html)
25 2. [Arduino v1.0.6 IDE](http://arduino.cc/en/main/software#toc2)
26 3. [Keyglove project source](https://github.com/jrowberg/keyglove) (or [full Github repo ZIP](https://github.com/jrowberg/keyglove/archive/master.zip))
27 4. [Teensyduino add-on for Arduino IDE](https://www.pjrc.com/teensy/td_download.html)
28 
29 ### Optional components:
30 
31 1. [GY-521 breakout board (MPU-6050)](http://www.amazon.com/Kootek-Arduino-MPU-6050-gyroscope-accelerometer/dp/B008BOPN40) for motion
32 2. [Bluegiga WT12 UART breakout board](http://www.inmojo.com/store/jeff-rowberg/item/wt12-uart-bluetooth-breakout-board) for Bluetooth
33 3. [I2Cdevlib project Arduino source](https://github.com/jrowberg/i2cdevlib/tree/master/Arduino) (or [full Github repo ZIP](https://github.com/jrowberg/i2cdevlib/archive/master.zip)) for motion sensor support code
34 4. [iWRAP project Arduino source](https://github.com/jrowberg/iwrap/tree/master/Arduino) (or [full Github repo ZIP](https://github.com/jrowberg/iwrap/archive/master.zip) for WT12 Bluetooth module support code
35 
36 The [Keyglove Kit PCB](https://www.facebook.com/media/set/?set=a.466087043425247.108696.160471600653461&type=3)
37 is currently undergoing some revisions and isn't available yet. When it is, it
38 will be very highly recommended, since it makes connecting sensors and modular
39 boards *significantly* easier.
40 
41 **NOTE!** The Teensy-based design currently requires the 3.3v modification done
42 with the MCP1725 regulator from PJRC (or elsewhere), since all of the motion and
43 wireless components that you may want to use here are also 3.3v systems. **MAKE
44 SURE YOU DO THIS**, or otherwise know exactly what you are doing.

46 # Firmware Preparation
47 
48 To get the Keyglove firmware running on your hardware, you will need to do a few
49 things first:
50 
51 1. Install the Arduino v1.0.6 IDE
52 2. Install the Teensyduino add-on
53 3. Clone or download and extract the Keyglove project source
54 4. **OPTIONAL:** Clone or download and extract the I2Cdevlib project source
55 5. **OPTIONAL:** Clone or download and extract the iWRAP project source
56 
57 If you have obtained the I2Cdevlib and/or iWRAP library sources, then make sure
58 you do the following as appropriate:
59 
60 1. Copy/move the `/i2cdevlib/Arduino/I2Cdev` folder to `~/Arduino/libraries/I2Cdev`
61 2. Copy/move the `/i2cdevlib/Arduino/MPU6050` folder to `~/Arduino/libraries/MPU6050`
62 3. Copy/move the `/iwrap/Arduino` folder to `~/Arduino/libraries/iWRAP` *(note renamed folder)*
63 
64 This will ensure that the required libraries are in the right spot so that the
65 Arduino IDE can find them. **If you already started the Arduino IDE by this
66 stage, you will need to close and re-open it in order for the new libraries to
67 be detected properly.**
68 
69 Once these steps are complete, all required software should be in place, and you
70 can continue with loading and configuring the firmware.

72 # Firmware Configuration
73 
74 One of the beautiful things about the Keyglove is that it is not just one single
75 device; it is actually a powerful, extensible platform that you can configure in
76 many different ways, in terms of both hardware and software. Although the entire
77 source tree may be edited, these are the files most likely to be of interest:
78 
79 - `config.h` - **Modular hardware/architecture configuration file**
80 - `application.cpp/.h` - **Custom application behavior definition**
81 - `application_stubs.h` - Custom application behavior stub/reference
82 - `custom_protocol.cpp/.h` - Custom protocol filtering and functional addition
83 - `version.h` - Firmware version info
84 
85 The first two are the most important, and you might not need the rest at all. In
86 any case, `config.h` is the first file that you should glance through to see if
87 anything needs to be changed. The current repo default assumes the following
88 configuration:
89 
90 - Teensy++ 2.0 board as the main host
91 - Single MPU-6050 motion sensor on the back of the hand
92 - USB connectivity for serial and keyboard/mouse HID
93 - No Bluetooth support
94 
95 You can change this using the various `#define`'d macros in `config.h`, if you
96 need. This is about the most basic configuration you would probably want for a
97 Keyglove (though you could go even further and remove the motion sensor). See
98 the documentation for KG_HOSTIF and KG_MOTION for detail.
99 
100 # Firmware Installation
101 
102 Now comes the moment of truth.
103 
104 1. Connect the Teensy++ 2.0 board via USB to your host
105 2. Start the Arduino IDE
106 3. Open the `keyglove.ino` file from the project source folder
107 4. From the "Tools" menu, select "Board" and then "Teensy++ 2.0"
108 5. From the "Tools" menu, select "Serial Port" and then the correct port for your Teensy
109 6. From the "Tools" menu, select "USB Type" and then "Serial + Keyboard + Mouse + Joystick" (or your choice)
110 7. From the "Tools" menu, select "CPU Speed" and then "8MHz" *(required after 3.3v modification)
111 8. Click the "Upload" button, or press Ctrl+U (or Command+U on a Mac)
112 
113 If everything went smoothly, it should compile without error and flash through
114 the Teensyduino add-on into your Teensy++ board. You should begin to see the LED
115 on the board blink briefly once every three seconds. If so, then it worked! You
116 can now open the Teensy's serial port and communicate using the KGAPI binary
117 protocol, or you can generate HID input reports by connecting certain sensors
118 together.
119 
120 The default firmware has very little internal autonomous behavior built in, but
121 this is easy to add. The following actions will be triggered with touches:
122 
123 - AY touch = motion sensor enabled
124 - AY release = motion sensor disabled
125 - DY touch = 'A' key pressed
126 - DY release = 'A' key released
127 - GY release = 'A' key pressed and released
128 
129 Internally, the "GY touch" action is of course detected, but it does not do
130 anything itself. The motion sensor will generate 100Hz data over the USB serial
131 port while it is active, but it will not move the mouse cursor (without some
132 modification to `application.h`). For reference:
133 
134 - The "Y" sensor is meant to go on the thumb tip
135 - The "A" sensor is meant to go on the index fingertip
136 - The "D" sensor is meant to go on the middle fingertip
137 - The "G" sensor is meant to go on the ring fingertip
138 - The "J" sensor is meant to go on the little fingertip
139 
140 In other words, the "AY" combination is touching the thumb tip to the index
141 fingertip, and the "DY" combination is touching the thumb tip to the middle
142 fingertip.
143 
144 The above simple set of actions gives you a basic functional reference to the
145 kinds of things you might do with the Keyglove's various sensors and its fully
146 event-driven functionality. More test software is currently under development
147 and will be published as soon as possible, with full source.
