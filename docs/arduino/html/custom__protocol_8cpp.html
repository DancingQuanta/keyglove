Detailed Description

USER-DEFINED: Custom protocol filtering implementations

Author
Jeff Rowberg
Date
2014-11-07

This file can be used to alter the behavior of predefined API packets without actually modifying the portions of code where those packets are defined. It can also be used to add support for new KGAPI commands which you may want for your custom functionality.

uint8_t filter_incoming_keyglove_packet ( uint8_t *  rxPacket)

Custom filter for incoming command packets.

Parameters
[in,out]rxPacketIncoming packet byte array (guaranteed to be well-formed)
Returns
KGAPI command packet fallthrough, zero allows and non-zero prevents

This function should not be used to add support for new commands; it is only for filtering/modifying packets on-the-fly if desired. To add new commands, refer to the process_protocol_command_custom() function instead.
